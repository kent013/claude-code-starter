# セッション書き出しスキル（session-export）

Claude Code がローカルに保管しているセッションファイル（会話履歴 `.jsonl`）から、利用者が指定したものを 1 つ特定して `workspace/sessions/` 以下にコピーする。

## 想定される呼び出し方

| 利用者の発言例 | 振る舞い |
|---|---|
| 「このセッションのログを報告用に出力して」「今のやりとりを書き出して」 | **直近のセッション** を 1 つ書き出す |
| 「doctor の話をしてたセッションを書き出して」「昨日の企画書の話のやつ送れるようにして」 | 候補を **1〜3 件提示** → 利用者が「それ」と答えたら確定書き出し |

1 回の実行で書き出すのは **1 セッションのみ**。複数まとめての出力は行わない。

---

## セッションファイルの保存場所

```
~/.claude/projects/<プロジェクトパスを - で繋いだ名前>/<UUID>.jsonl
```

例: `/Users/ishitoya/repository/claude-code-starter` → `~/.claude/projects/-Users-ishitoya-repository-claude-code-starter/`

---

## 動作フロー

### Step 1: 現在のプロジェクトを特定

```sh
pwd
```

取得したパスを `/` → `-` に置換し、`~/.claude/projects/<slug>/` の存在を確認（`ls ~/.claude/projects/<slug>/` で 1 つ以上の `.jsonl` があるか）。

無い場合:
- 「このプロジェクトではまだ Claude Code のセッション履歴が見つかりませんでした」と報告して終了

### Step 2: 依頼タイプを判定

利用者の発言から 2 パターンに分岐:

#### パターン A — 「このセッション」「今のやりとり」「直近」

**直近のセッション** を直接対象にする（選択ステップを省く）:

```sh
ls -t ~/.claude/projects/<slug>/*.jsonl
```

の先頭 1 件を選択し、Step 4 へ進む（選択確認は不要）。

> **補足**: 実行中の会話自身に対応する .jsonl は、ほぼ最新の mtime で特定できる。ただし会話中も更新され続けているため、コピー時点のスナップショットになる点を完了報告に添える。

#### パターン B — キーワード・日付等での指定

利用者発言から検索条件を抽出（例: 「doctor」「企画書」「昨日」）。

1. 対象ディレクトリの `.jsonl` を mtime 新しい順に全件取得
2. 各ファイルの中身を **Grep ツール** で検索（日本語キーワード含む）。件数が多い場合は直近 20 件程度に絞ってよい
3. 日付指定なら mtime でさらに絞り込み
4. **上位 1〜3 件** を利用者に提示:

```
以下が候補です。どれが該当しますか？

1. 2026-04-15 14:12 (128 行)
   最初: 「doctorして」
   発言1: 「結果をワークスペースに保存しておいて」

2. 2026-04-14 20:30 ( 42 行)
   最初: 「Codex で doctor のスクリプトを書きたい」

3. 2026-04-14 09:05 ( 18 行)
   最初: 「doctor スキルの許可ダイアログを減らしたい」

番号で教えてください。
```

利用者の回答（「1」「それ」「一番上」等）で確定。曖昧な場合は聞き直す。

候補が 0 件なら「該当するセッションが見つかりませんでした。別の言い回しで試してみますか？」と返す。

### Step 3: 対象セッションのメタ情報を取得

確定したセッションファイルについて:

- UUID（ファイル名）
- mtime（作成時刻）
- 行数（メッセージ総数の目安）: `wc -l`
- 最初のユーザー発言の冒頭（100 文字程度）
- 最後のユーザー発言の冒頭（100 文字程度）

これらは Read ツールで .jsonl の先頭・末尾を読んで抽出する。

### Step 4: 書き出し先ディレクトリを作成

```sh
mkdir -p workspace/sessions/<YYYYMMDD-HHMMSS>
```

タイムスタンプは `date +%Y%m%d-%H%M%S` を 1 回実行して取得。

### Step 5: 対象 .jsonl をコピー

```sh
cp <source .jsonl> workspace/sessions/<YYYYMMDD-HHMMSS>/
```

元ファイル名（`<UUID>.jsonl`）を保持する。

### Step 6: サマリファイル `README.md` を同梱

`workspace/sessions/<YYYYMMDD-HHMMSS>/README.md` を **Write ツール**で作成:

```markdown
# セッション書き出し — YYYY-MM-DD HH:MM:SS

| 項目 | 内容 |
|---|---|
| ファイル | <UUID>.jsonl |
| 元の作成日時 | 2026-04-15 14:12 |
| 行数 | 128 |
| 最初の発言 | 「doctorして」 |
| 最後の発言 | 「commit pushして」 |

## 元パス

`~/.claude/projects/<slug>/<UUID>.jsonl`

## 注意

- この `.jsonl` には会話内容・ツール呼び出し・ツール出力がすべて含まれます
- **外部に送る前に中身を確認してください**（認証トークン・個人情報・社内情報などが含まれていないか）
- 気になる行があれば削除してから送付するか、送付そのものを取りやめてください
```

### Step 7: 完了報告

```
書き出しが完了しました。

保存先: workspace/sessions/<YYYYMMDD-HHMMSS>/
- <UUID>.jsonl（<行数> 行、元の作成日時: <mtime>）
- README.md（メタ情報）

報告のために送る前に、中身を一度ご確認ください。
機密情報・認証トークンが混ざっている可能性があります。
```

実行中セッション自身を書き出した場合は、末尾に一言添える:

```
※ 今ご覧になっている会話自身のログです。会話継続中の時点でのスナップショットなので、
  この後のやりとりは含まれません。必要ならまた実行してください。
```

## プライバシー注意（重要）

セッションファイルには以下が含まれる可能性があります:

- 利用者の発言全文
- 貼り付けられたコード・ファイル内容
- ツール出力に含まれる環境情報（パス、認証状態など）
- 社内情報・プロジェクト固有情報

**利用者に外部送付前の中身確認を毎回促す**こと。スキル自身では中身のマスキング・加工はしない（誤検知による欠落を避けるため）。

---

## 動作原則

- 対象は **現在のプロジェクトのセッションのみ**（他プロジェクトのセッションは原則扱わない。明示的に絶対パスで指定された場合のみ例外対応）
- **コピーのみ**（元ファイルは絶対に変更・削除しない）
- **1 実行 = 1 セッション**。複数まとめ出力・全件出力は行わない
- `workspace/sessions/` 配下以外には何も書かない
- セッション内容をスキル自身が要約・翻訳することはしない

---

## 許可設定との対応

このスキルで使う Bash コマンドは [.claude/settings.local.json](.claude/settings.local.json) で事前許可されている想定:

- `pwd`, `ls`, `date`, `wc` — 一覧・メタ情報
- `cp` — セッションファイルコピー（源が `~/.claude/projects/**`、先が `workspace/sessions/**`）
- `mkdir -p workspace/**`, `Write(workspace/**)` — 保存先作成・サマリ生成

許可が通らずダイアログが出た場合は、利用者に `.claude/settings.local.json` への追加を案内してください（該当パターンを 1 行追加するだけ）。
